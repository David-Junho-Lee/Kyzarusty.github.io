Problem Solving

Blocked on a simple problem

I had a problem with the 'deBee' function in the Built-in Methods kata. I was to make the words in an array all lowercase so I could filter them all out. I knew I was close to the solution, but the function wouldn't work. I used Pseudocode, console logging, and googling to find out what I was doing wrong, but they all failed to reach the solution. I was frustrated because I kept googling and trying to alter my code, but none of them worked. I finally gave up on solving it myself and seek for help on the discord tech-questions channel. There was already athread on the problem, and I got a hint from the thread and reached the solution. I've learned that there is always a solution to a problem. Even if you can't solve it yourself, help is around you; you just need to seek it.

Reflect on how confident you feel using each of these problem-solving techniques/processes

Pseudocode

Pseudocode is very useful, it helps me to focus on small pieces of big puzzle, so I don't get lost during process.
It is relatively new to me and I'm not completely confident using it, but it definitely help me to build code step by step.


Trying something

I use this technique most of the time as I want to solve problems on my own.
Trying something helps and works but I learnt that I need to stop being stubborn and start asking questions more often.


Rubber ducky method

I'm not really a big fan of rubber ducky method, but I use similar technique which is talking to myself.
Talking to myself about problems helps me understand them better.


Reading error messages

Reading error messages can be a great help if the error messages directly informs me what I'm missing.
Even a simple syntax error message help me realize what I did wrong.


Console logging

My favourite type of problem solving techniques, as I can use it as a tool to display the outcomes.
I use it all the time to see where I'm at with the problem.


Googling

Googling is a god-send to problem solving when it comes to coding.
When I come across new feature, I always google to find out and understand it before I began to try solve problems.


Asking your peers/facilitators for help

This is something I lack, I'm too focused on solving problems on my own.
If I'm stuck after trying many other problem solving techniques, I then seek for help, asking in discord.


Improving your process with reflection

Even if I solved a problem using all the techniques mentioned above, if I don't understand how,
then I can't move on to next problem. I learn the most by reflecting process of solving problem.
